# RepoShield

> `git clone` 하기 **전에** GitHub 저장소의 안전성을 검증하는 무료 웹 도구 — by BSD (Business System Development)

GitHub URL만 입력하면, 코드를 내려받기 전에 **API 키 탈취·스미싱·악성 설치 스크립트·원격 코드 실행** 같은 위험 패턴을 정적 분석하고, 비전문가도 3초 안에 판단할 수 있게 **신호등(안전/주의/위험)** 으로 보여줍니다. 백엔드 없이 **브라우저에서 GitHub 공개 API로 직접** 동작하므로 그대로 정적 호스팅에 배포할 수 있습니다.

## 주요 기능

- **clone 전 보안 검증** — 저장소 파일을 정적 스캔해 위험 패턴 탐지 (코드를 실행하지 않으므로 분석 자체가 안전)
- **4축 점수 + 원형 그래프** — 신뢰도 · 안정성 · 적합성 · 위험도를 동적으로 시각화
- **신호등 판정 + 전체화면 경고** — 절대 clone하면 안 되는 수준이면 강한 경고
- **외부 인텔리전스 강화**
  - 의존성 공급망 검사 (OSV.dev / npm / PyPI) — **악성 패키지**(예: 멀웨어 주입)와 **일반 CVE(공급망 위생)** 를 구분
  - 출처 불일치 감지 ("GitHub ≠ 게시 패키지")
  - 커밋·소유자 포렌식 (신생 위장 저장소 탐지)
  - 커밋된 시크릿 · 엔트로피 기반 난독화 탐지
  - 이미 보고된 보안 권고(GitHub Advisories / OSV) 표시
- **AI 코드 해설 (선택)** — 본인 OpenRouter 키로 의심 코드를 자연어로 설명
- **트렌딩 런처** — 오픈소스 트렌드 사이트 모음 (무한 슬라이드)

## 설계 원칙

- **안전성 제1원칙** — "검사하지 못한 것"을 "안전"으로 표시하지 않습니다. 스캔 한계를 항상 정직하게 표기합니다.
- **거짓 양치기 금지** — "의존성에 알려진 CVE가 있다"(위생)와 "이 저장소가 악성이다"(악성 의도)를 분리합니다. 일반 CVE는 전체화면 경고를 띄우지 않습니다.
- **키를 새지 않는 도구** — 입력한 GitHub 토큰·AI 키는 브라우저에만 머물며, GitHub/OpenRouter 외 어디로도 전송되지 않습니다.

> ⚠️ RepoShield의 판정은 휴리스틱 정적 분석 결과이며 100% 보장이 아닙니다. 최종 판단은 사용자에게 있습니다.

## 기술 스택

React + Vite (단일 페이지) · 순수 CSS · 백엔드 없음.

## 개발

```bash
npm install
npm run dev      # 개발 서버 (http://localhost:5173)
npm run build    # 프로덕션 빌드 (dist/)
npm run preview  # 빌드 미리보기
```

GitHub 미인증 API는 시간당 60회로 제한됩니다. 검증창에서 **권한 없는(read-only) GitHub 토큰**을 넣으면 5,000회/시간으로 완화됩니다 (토큰은 선택적으로 이 기기에만 저장).
